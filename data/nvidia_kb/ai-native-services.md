# AI-native Services — conceito e critérios

## A tese (Sequoia + Emergence)
- **Sequoia — "Services as the New Software"**: a IA permite que software passe a vender não só
  ferramentas, mas **trabalho e resultados (outcomes)**, expandindo o mercado endereçável do
  tamanho do software para o tamanho dos **serviços**. (https://sequoiacap.com/article/services-the-new-software/)
- **Emergence — "The AI-native Services Playbook"**: empresas AI-native combinam software +
  agentes + dados proprietários + serviço especializado para **entregar o resultado de ponta a
  ponta**, e não apenas uma ferramenta.

## AI-native vs AI-enabled vs non-AI
- **AI-native**: a IA é o núcleo do valor; a empresa **produz IA** (treina/post-traina modelos),
  possui **dados proprietários**, **otimiza a stack** (custo/latência), tem **workflow profundo**
  e **defensibilidade** frente aos grandes labs. (A pergunta "por que nem toda startup é como a
  Tractian?" é exatamente sobre isto.)
- **AI-enabled**: usa IA de forma relevante, mas majoritariamente via **APIs externas**, com
  menos dados proprietários e menos otimização técnica.
- **non-AI**: IA é periférica ou inexistente.

## Critérios para qualificar AI-native (usados no scoring)
1. **Dados proprietários** — possui dados únicos que alimentam um data flywheel.
2. **Engenharia de modelos** — treina, fine-tuna ou faz **post-training** (sinal forte).
3. **Otimização de inferência** — faz engenharia eficiente de custo/latência (não só API).
4. **Profundidade de workflow** — entrega o resultado, não só uma ferramenta.
5. **IA no produto** — IA é central ao valor entregue.
6. **Defensibilidade** — moat frente à comoditização pelos grandes labs (OpenAI, Anthropic, Google).

## O risco de "wrapper" (Lab Displacement Risk)
Startups que **apenas** conectam uma API de LLM a uma interface, sem dados próprios, workflow
profundo ou otimização, podem ser substituídas por funcionalidades nativas dos grandes labs.
Quanto menor a defensibilidade e maior a dependência de API genérica, **maior o risco de
deslocamento**. A oportunidade da NVIDIA é ajudar essas empresas a subir a stack e se
diferenciarem.
